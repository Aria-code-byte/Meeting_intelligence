# AI 会议内容理解助手 - CLI 使用指南

一个简洁的命令行工具，用于会议内容的结构化总结。

## 功能特性

- **模板管理**: 创建、查看、删除自定义模板
- **模拟上传**: 模拟音频文件上传（带进度条）
- **模拟转写**: 模拟语音识别生成文字稿
- **智能总结**: 基于模板生成结构化会议总结
- **数据持久化**: 使用 JSON 文件存储模板配置
- **异常处理**: 优雅处理文件不存在或 JSON 格式错误

## 快速开始

### 运行 CLI

```bash
python -m meeting_intelligence.cli
```

或直接运行：

```bash
python3 meeting_intelligence/cli.py
```

### 主菜单

```
==================================================
  AI 会议内容理解助手
==================================================

请选择操作：
  1. 上传音频（模拟）
  2. 生成文字稿（模拟）
  3. 生成会议总结
  4. 模板管理
  5. 退出
```

## Prompt 构建逻辑

`build_prompt()` 函数按照以下格式拼接：

```
你是一名专业会议分析助手。请从【{模板名称}】的视角总结以下会议内容。
总结要求：{模板描述}。请使用结构化方式输出。
会议内容：{转录文本}
```

## 数据存储

模板数据存储在项目根目录的 `templates.json` 文件中：

```json
{
  "version": "1.0",
  "templates": [
    {
      "name": "通用总结",
      "description": "请提供会议的全面总结..."
    },
    {
      "name": "大学生视角",
      "description": "从大学生学习的角度总结会议内容..."
    }
  ]
}
```

## 默认模板

工具初始化时会创建两个默认模板：

### 1. 通用总结

适用于一般场景，提供全面总结。

### 2. 大学生视角

从学习角度总结，突出知识点和应用价值。

## 技术特点

- **面向对象设计**: 使用类封装功能，职责清晰
- **数据持久化**: JSON 格式存储，便于备份和迁移
- **异常处理**: 优雅处理各种错误情况
- **进度展示**: ASCII 进度条提升用户体验
- **扩展性**: 预留 LLM API 接口，易于集成
