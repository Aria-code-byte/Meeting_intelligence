# 技术实现

## 1. 技术架构概述

本项目采用**模块化、分层式**技术架构，实现从音视频输入到结构化会议输出的端到端智能处理流程。

### 1.1 架构设计原则

| 原则 | 说明 | 实现方式 |
|------|------|---------|
| **单一真理来源** | ASR 原始转录不可覆盖 | 分层工件存储架构 |
| **防御性编程** | 完整的输入验证和错误处理 | 类型注解、边界检查、回退机制 |
| **渐进式增强** | 基础功能独立运行，高级功能可选降级 | 模块解耦、可选特性开关 |
| **测试驱动** | 454+ 测试用例覆盖核心流程 | pytest 单元测试、集成测试 |

## 2. 核心技术模块

### 2.1 音视频处理模块

**技术选型**: FFmpeg

**功能特性**:
- 音频提取: 从 MP4/MKV/MOV 等视频格式中提取音轨
- 格式标准化: 统一转换为 WAV, 16kHz, 单声道, 16-bit PCM
- 音量归一化: 自动调节音频电平，提升 ASR 准确度
- 静音检测: 可选的静音片段裁剪

**技术实现**:
```python
# 使用 FFmpeg subprocess 调用实现音频处理
# -ar 16000: 采样率 16kHz
# -ac 1: 单声道
# -acodec pcm_s16le: 16-bit PCM 编码
```

**性能指标**:
- 处理速度: ~1.5x 实时速度（10分钟视频 ~7分钟处理）
- 支持格式: MP3, WAV, M4A, MP4, MKV, MOV, AVI
- 文件大小上限: 2.5GB

### 2.2 ASR 语音转文字模块

**技术选型**: Whisper + faster-whisper

| 实现方式 | 速度 | 内存占用 | 准确度 | 使用场景 |
|---------|------|---------|--------|----------|
| openai-whisper | 1x | ~2GB | 95% (medium) | 开发测试 |
| **faster-whisper** | **4-5x** | **~1GB** | **95%** | **生产环境** |

**核心功能**:
- 多语言支持: 中英文混合识别，自动语言检测
- 时间戳精度: 句子级别起止时间（浮点秒，精确到毫秒）
- 智能时间戳修正: 自动裁剪超出范围的时间戳，过滤异常 utterance

**技术实现**:
```python
class FasterWhisperProvider(BaseASRProvider):
    def __init__(self, model_size="medium", device="auto", compute_type="default"):
        # 自动选择计算精度：GPU用float16，CPU用int8
        self._model = WhisperModel(model_size, device, compute_type)

    def transcribe(self, audio_path, language="auto"):
        segments, info = self._model.transcribe(
            audio_path,
            language=language_param,
            vad_filter=True,  # 语音活动检测过滤静音
            beam_size=5,      # 束搜索大小
        )
```

**性能指标**:
- 处理速度: medium 模型 ~0.15x 实时速度（10分钟音频 ~10分钟处理）
- 准确率: medium 模型 95%+ (中文语音)
- 并发支持: 单进程顺序处理（可扩展为多进程）

### 2.3 转录增强模块

**技术选型**: LLM (GPT-4/Claude/GLM/DeepSeek)

**核心创新**: 确定性分块 + 单句级别增强

#### 2.3.1 确定性分块系统

| 参数 | 值 | 说明 |
|------|-----|------|
| 窗口大小 | 60秒 | 固定时间窗口 |
| 重叠时间 | 10秒 | 保证上下文连续性 |
| 时间模型 | 整数毫秒 | 避免浮点精度问题 |

**技术实现**:
```python
class UtteranceChunker:
    def build_fixed_chunks(self, duration_ms, window_seconds=60, overlap_seconds=10):
        # 生成确定性的时间窗口
        window_size_ms = window_seconds * 1000
        overlap_ms = overlap_seconds * 1000

        chunks = []
        start_ms = 0
        while start_ms < duration_ms:
            end_ms = min(start_ms + window_size_ms, duration_ms)
            chunks.append(TimeWindow(start_ms, end_ms))
            start_ms = end_ms - overlap_ms
```

#### 2.3.2 单句级别语义增强

**三级映射策略**:
1. **精确匹配**: 字符串完全一致 → 置信度 1.0
2. **Embedding 相似度**: 向量相似度匹配 → 置信度 0.0-1.0
3. **位置映射**: 时间戳邻近匹配 → 置信度 0.0-0.5

**多特征置信度计算**:
```python
confidence = (
    0.4 * similarity_score +      # 语义相似度
    0.3 * position_score +        # 位置接近度
    0.2 * length_ratio +          # 长度相似度
    0.1 * edit_distance_score     # 编辑距离
)
```

**智能回退机制**:
- 单句 LLM 失败 → 使用原始句子
- 整块失败 → 逐句回退到原始
- 置信度过低 → 混合使用增强和原始

**性能指标**:
- 处理速度: ~500字/分钟 (GPT-4o-mini)
- 准确提升: 错别字修正率 95%+
- 语义保真: 98%+ 内容一致性

### 2.4 会议总结模块

**技术选型**: 多 LLM 提供商支持

**模板系统**:
- 预设模板: 通用周报、技术评审、高管汇报等
- 自定义模板: 用户可创建角色视角模板
- Few-shot 提示: 带示例的提示词模板

**技术实现**:
```python
class LLMTranscriptEnhancer:
    def enhance(self, transcript_text: str, template: str = "general"):
        prompt = self._build_prompt(template, transcript_text)
        response = self.llm_client.generate(
            messages=[LLMMessage(role="user", content=prompt)],
            temperature=0.3,
            max_tokens=4000
        )
        return EnhancedTranscriptResult(
            enhanced_text=response.content,
            source_text=transcript_text,
            template_used=template
        )
```

**支持的模板**:
| 模板名称 | 适用场景 | 特点 |
|---------|---------|------|
| general | 通用会议 | Few-shot 示例，平衡优化 |
| technical | 技术评审 | 保留专业术语，结构化输出 |
| executive | 高管汇报 | 商务风格，简洁明了 |
| minimal | 最小改动 | 仅修正明显错误 |
| speech-to-text-refiner | 个人叙述 | 加粗数字，精修表达 |

## 3. 数据模型与存储

### 3.1 核心数据结构

```python
@dataclass
class Utterance:
    """单条语音识别结果"""
    start: float      # 开始时间（秒）
    end: float        # 结束时间（秒）
    text: str         # 识别文本

@dataclass
class TranscriptionResult:
    """语音识别结果接口"""
    utterances: List[Utterance]
    audio_path: str
    duration: float
    output_path: str
    asr_provider: str
    timestamp: str
    transcript_path: Optional[str]           # 原始转录（始终指向 raw ASR）
    enhanced_transcript_path: Optional[str]  # 增强版转录（预留）
```

### 3.2 分层工件存储

**架构原则**: Raw 不可覆盖，Enhanced 独立存储

```
data/
├── transcripts/           # ASR 原始转录（不可修改）
│   └── transcript_*.json
├── outputs/              # 增强和衍生工件
│   ├── *_enhanced_*.txt  # LLM 增强转录
│   └── *_refined_*.txt   # 时间索引实录
└── summaries/            # 会议总结
    └── summary_*.json
```

### 3.3 数据库设计 (Web 后端)

|------|------|---------|
| meetings | 会议记录 | id, title, video_path, status, progress |

## 5. 性能优化策略

### 5.1 ASR 加速

| 策略 | 效果 | 实现方式 |
|------|------|---------|
| faster-whisper | 4-5x 加速 | CTK 实现，优化的模型推理 |
| VAD 过滤 | 减少 20-30% 处理时间 | 语音活动检测跳过静音 |
| 模型选择 | 2-3x 速度差异 | 根据精度需求选择模型大小 |

### 5.2 LLM 调用优化

| 策略 | 效果 | 实现方式 |
|------|------|---------|
| 分块处理 | 降低 token 消耗 | 60秒窗口分块，减少上下文长度 |
| 模型选择 | 10x 成本差异 | GPT-4o-mini vs GPT-4 |
| 温度控制 | 提高稳定性 | temperature=0.3 固定输出 |
| 并发请求 | 线性加速 | 多块并行 LLM 调用 |

### 5.3 缓存策略

- ASR 结果缓存: 相同音频文件不重复转写
- LLM 响应缓存: 相同输入复用输出
- 模型缓存: Whisper 模型常驻内存

## 10. 总结

本项目采用**模块化、分层式**技术架构，实现了从音视频输入到结构化会议输出的端到端智能处理。核心技术创新包括：

1. **确定性分块系统**: 固定窗口 + 重叠策略，保证上下文连续性
2. **单句级别语义增强**: 三级映射策略，实现精准对齐和智能回退
3. **faster-whisper 集成**: 4-5x 速度提升，内存占用减半
4. **CLI 逻辑强制对齐**: Web 与 CLI 共享核心逻辑，确保一致性

通过 454+ 测试用例覆盖和防御性编程实践，确保系统稳定性和可维护性。技术架构具有良好的可扩展性，支持多 LLM 提供商、多 ASR 引擎和灵活的部署方式。
