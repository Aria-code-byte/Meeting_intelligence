## 项目简介

AI 会议智能助手是一个基于人工智能的会议处理系统，能够将音频/视频会议自动转换为：
- **原始会议文档**（逐字转写 + 精确时间轴）
- **增强版转录**（LLM 修正错别字、优化语句流畅度）
- **带时间索引的结构化实录**（分块整理，可定位播放）
- **自定义模板总结**（支持技术/高管/项目等多种场景）

### 核心功能

| 功能模块 | 说明 |
|----------|------|
| 📤 **会议上传** | 支持音频（MP3/WAV/M4A）和视频（MP4/AVI/MOV）文件 |
| 🎙️ **智能转录** | WhisperX + pyannote 说话人分离 |
| 🤖 **AI 总结** | 支持 DeepSeek/OpenAI/Anthropic/Ollama |
| ✏️ **在线编辑** | 标题、总结、行动项可视化编辑 |
| 📤 **多格式导出** | Markdown/TXT/JSON 一键导出 |
| ⚙️ **模板管理** | 自定义总结模板，适配不同场景 |

### 技术亮点

- **454+ 测试用例全部通过** - 代码质量保障
- **模块化架构** - 前后端分离，易于扩展
- **多 LLM 支持** - 兼容国产大模型（DeepSeek/智谱）
- **说话人分离** - 自动识别不同发言人
- **词级时间戳** - 精确到每个词的时间定位

## 安装步骤

### 第一步：安装 FFmpeg

FFmpeg 用于音视频处理和格式转换。

**Windows:**
```bash
# 下载 FFmpeg
# 1. 访问 https://www.gyan.dev/ffmpeg/builds/
# 2. 下载 ffmpeg-release-essentials.zip
# 3. 解压到 C:\ffmpeg
# 4. 添加 C:\ffmpeg\bin 到系统环境变量 PATH

# 验证安装
ffmpeg -version
```

**macOS:**
```bash
brew install ffmpeg
ffmpeg -version
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get update
sudo apt-get install ffmpeg
ffmpeg -version
```

### 第二步：克隆项目

```bash
# 克隆项目到本地
git clone <项目地址>
cd Meeting_intelligence
```

### 第三步：后端安装

```bash
# 1. 创建 Python 虚拟环境（推荐）
python3 -m venv .venv

# 2. 激活虚拟环境
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

# 3. 安装后端依赖
pip install -r backend/requirements.txt

# 4. 配置环境变量
cp backend/.env.example backend/.env
```

### 第四步：前端安装

```bash
# 1. 进入前端目录
cd web_backend/react-ui

# 2. 安装依赖
npm install

# 3. 返回项目根目录
cd ../..
```

## 配置说明

### 环境变量配置

编辑 `backend/.env` 文件，根据需要配置 AI Provider。

#### 方案一：DeepSeek（推荐，性价比高）

```bash
# 后端 API 配置
AI_SUMMARY_PROVIDER=backend
LLM_PROVIDER=openai
OPENAI_API_KEY=<你的DeepSeek_API_Key>
OPENAI_BASE_URL=https://api.deepseek.com/v1
OPENAI_MODEL=deepseek-chat
```

> 获取 DeepSeek API Key: https://platform.deepseek.com/

#### 方案二：Fallback 模式（无需 API Key）

```bash
AI_SUMMARY_PROVIDER=fallback
```

#### 方案三：OpenAI 官方

```bash
AI_SUMMARY_PROVIDER=backend
LLM_PROVIDER=openai
OPENAI_API_KEY=<你的OpenAI_API_Key>
OPENAI_MODEL=gpt-4o-mini
```

### Hugging Face Token（说话人分离）

如需使用说话人分离功能，需要配置 Hugging Face Token：

```bash
# 在 backend/.env 中添加
HF_TOKEN=<你的HuggingFace_Token>
```

> 获取方式: https://huggingface.co/settings/tokens
> 需要接受 pyannote/speaker-diarization-community-1 的使用条款

## 常见问题

### Q1: FFmpeg 未找到

**问题**: `ffmpeg: command not found`

**解决**: 
- 确认 FFmpeg 已正确安装
- 检查系统环境变量 PATH 是否包含 FFmpeg 路径
- 重启终端使环境变量生效

### Q2: 端口被占用

**问题**: `Address already in use`

**解决**:
```bash
# Windows - 查找并结束占用端口的进程
netstat -ano | findstr :8000
taskkill /PID <进程ID> /F

# Linux/macOS
lsof -ti:8000 | xargs kill -9
```

### Q3: Python 依赖安装失败

**问题**: pip install 报错

**解决**:
```bash
# 升级 pip
python -m pip install --upgrade pip

# 使用清华镜像源
pip install -r backend/requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q4: Whisper 模型下载慢

**问题**: 首次运行时模型下载时间长

**解决**:
- 确保网络连接稳定
- 或使用已下载的模型文件
- 或使用 faster-whisper 替代

### Q5: LLM API 调用失败

**问题**: 总结生成失败

**解决**:
- 检查 API Key 是否正确
- 确认账户有足够的额度
- 检查网络连接
- 查看后端日志获取详细错误信息

### Q6: 前端无法连接后端

**问题**: 前端显示 "连接失败"

**解决**:
- 确认后端服务正在运行（访问 http://localhost:8000/api/v1/health）
- 检查 Vite proxy 配置
- 查看浏览器控制台错误信息
