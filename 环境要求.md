## 1. 硬件要求

### 最低配置

| 组件 | 要求 | 说明 |
|------|------|------|
| **CPU** | 双核 2.0GHz+ | 运行 Whisper 模型需要较强 CPU |
| **内存** | 8 GB RAM | Whisper 模型加载需要约 2-4GB 内存 |
| **磁盘** | 5 GB 可用空间 | Python 环境、Node 模块、模型文件 |
| **网络** | 宽带连接 | 下载依赖和 AI API 调用 |

### 推荐配置

| 组件 | 要求 | 说明 |
|------|------|------|
| **CPU** | 四核 3.0GHz+ | 更快的语音识别处理 |
| **内存** | 16 GB RAM | 流畅运行所有服务 |
| **磁盘** | 10 GB 可用空间 | 包含模型缓存和日志文件 |
| **GPU** | NVIDIA CUDA（可选） | 加速 Whisper 模型推理 |

### 特殊说明

- **Whisper 模型大小**：
  - `tiny`: ~40 MB
  - `base`: ~140 MB
  - `small`: ~460 MB
  - `medium`: ~1.4 GB
  - `large-v3`: ~2.9 GB

- **首次运行**：会自动下载默认 Whisper 模型（~140 MB），需要稳定的网络连接

## 3. 软件依赖

### 必需软件

#### Python

| 项目 | 要求 |
|------|------|
| **版本** | 3.10 或更高（推荐 3.12） |
| **下载** | https://www.python.org/downloads/ |
| **验证** | `python --version` |

**安装注意事项**：
- ✅ 安装时勾选 "Add Python to PATH"
- ✅ Windows 用户选择 "Install for all users"

#### Node.js

| 项目 | 要求 |
|------|------|
| **版本** | 18.0 或更高（推荐 20.x LTS） |
| **下载** | https://nodejs.org/ |
| **验证** | `node --version` |

**安装注意事项**：
- ✅ 会自动安装 npm（Node 包管理器）
- ✅ 推荐下载 LTS（长期支持）版本

#### FFmpeg

| 项目 | 要求 |
|------|------|
| **版本** | 4.0 或更高 |
| **用途** | 音视频格式转换和处理 |
| **验证** | `ffmpeg -version` |

**安装方式**：

**Windows:**
```bash
# 1. 下载 FFmpeg
https://www.gyan.dev/ffmpeg/builds/ffmpeg-release-essentials.zip

# 2. 解压到 C:\ffmpeg

# 3. 添加到环境变量
# 系统属性 → 环境变量 → 系统变量 → Path → 新建
# 输入: C:\ffmpeg\bin

# 4. 验证安装
ffmpeg -version
```

**macOS:**
```bash
# 使用 Homebrew 安装
brew install ffmpeg

# 验证安装
ffmpeg -version
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get update
sudo apt-get install ffmpeg

# 验证安装
ffmpeg -version
```

| 内容 | 大小 | 时间（参考） |

## 8. 常见环境问题

### Python 版本问题

```bash
# 检查 Python 版本
python --version

# 如果版本过低，请访问
# https://www.python.org/downloads/
```

### Node.js 版本问题

```bash
# 检查 Node.js 版本
node --version

# 如果版本过低，请访问
# https://nodejs.org/
```

### FFmpeg 找不到

```bash
# 检查 FFmpeg 是否安装
ffmpeg -version

# Windows: 确认 C:\ffmpeg\bin 在 PATH 中
# macOS: 使用 brew install ffmpeg
# Linux: sudo apt-get install ffmpeg
```

### 端口被占用

```bash
# Windows - 查找占用进程
netstat -ano | findstr :8000

# Linux/macOS - 查找占用进程
lsof -ti:8000

# 解决方案: 关闭占用端口的程序
```
